# VisiBill — Frontend Design Specification

This document defines the visual design system for VisiBill. Every component, page, and layout created by Claude Code must follow these rules.

---

## 1. Design Identity

**Vision:** A professional, information-dense fintech CRM inspired by the Twilio CRM template — white sidebar, warm orange top nav, master-detail layouts, and tabbed information architecture — adapted for construction finance transparency. Every screen should look like it was designed by a product team at a real fintech company, not generated by AI.

**Core Principles:**
- **White sidebar + dark orange top nav:** The sidebar is white with a subtle right border. The top navigation bar uses a warm, muted orange (`#D4792A`) — rich but not harsh. All content areas are pure white.
- **Master-detail pattern:** List views always use a left panel (scrollable list with search + filter pills) and a right detail panel (tabbed sections with field grids). This is the primary navigation pattern for Projects, Clients, and Transactions.
- **Single accent color:** Peach/orange (`#E7651C`) is the only accent. Replaces Twilio's blue across all active states, CTAs, and interactive elements.
- **Filter pill tabs:** Horizontal pill-shaped toggle buttons for filtering lists (All / Active / Completed / Pending). Active state = filled peach with border.
- **Data density over decoration:** Field grids display information in labeled pairs (bold label above, value below) in 2-3 column layouts. No icons next to field labels.
- **Trust-first:** This platform handles real money. Every pixel reinforces professionalism, reliability, and transparency.

**What This Is NOT:**
- Not Salesforce Cosmos style (no grey canvas, no floating elevated cards)
- Not dark-themed (no dark navy sidebar, no dark backgrounds)
- Not a consumer app (no gamification, no badges, no streaks)
- Not multi-color (no green accents, no blue accents — peach/orange family only)
- Not icon-heavy (icons in sidebar nav only, not decorating section headers or field labels)
- **NOT vibe-coded.** No hero sections with gradient blobs. No oversized padding with 3 words per screen. No glassmorphism, frosted glass, or backdrop-blur effects. No trendy oversized rounded corners (max `rounded-lg`). No generic "modern SaaS" aesthetic with massive whitespace and centered single-column layouts. No AI-slop patterns: purple gradients, floating cards on pastel backgrounds, decorative illustrations, or "clean and minimal" emptiness. This is a dense, professional, data-first CRM — every pixel shows real information.

### Palette Tokens (tailwind.config.ts)

| Token | Hex | Usage |
|-------|-----|-------|
| `off-black` | `#170B01` | All body text |
| `peach-50` | `#FFF8F2` | Info banner backgrounds, hover tints |
| `peach-100` | `#FFEAD6` | Light accent backgrounds, selected list items |
| `peach-600` | `#D4792A` | Top nav background AND warning status text (one token, two uses) |
| `peach-700` | `#C06E24` | Top nav button hover states |
| `peach-800` | `#8B4513` | Critical emphasis text |

### Color Usage Rules

- **Primary orange (`#E7651C`):** Sidebar active icon, primary CTA buttons, filter pill active state, progress bar fills, status badge accents.
- **Warm burnt orange (`#D4792A`):** Top navigation bar background only. A muted, professional orange — not neon, not the same as primary `#E7651C`. Slightly darker and more golden.
- **Off-black (`#170B01`):** All text — headings, labels, values, descriptions.
- **Sub-text:** `text-off-black/40` for metadata and secondary information only.
- **Info banners:** `bg-peach-50` with `border-l-4 border-primary` and peach icon. Replaces Twilio's blue info banners.
- **Status colors:** All within peach/orange family. Normal = `bg-primary/20 text-primary`, Warning = `bg-primary text-white`, Critical = `bg-peach-800 text-white`.
- **Destructive:** Burnt orange (`--destructive`), not red. Delete buttons use outlined destructive style.
- **Page background:** `bg-white` — NOT grey. The Twilio template uses white content areas.
- **Selected list item:** `bg-peach-100` with `border-l-4 border-primary`. Replaces Twilio's blue selection highlight.

### List Panel (Left)
- Width: `w-[380px]` fixed, `border-r border-border`
- Header: page title (`text-2xl font-bold`) + count (`text-sm text-off-black/40`)
- Search: rounded input with search icon, full width, `mt-4`
- Filter pills: horizontal row of pill buttons below search
  - Default: `border border-border rounded-full px-3 py-1 text-sm`
  - Active: `bg-primary text-white border-primary rounded-full`
  - Optional: filter/sort icon button at end of row
- List items: vertically scrollable
  - Avatar circle (initials, `bg-peach-100 text-primary`) + name (bold) + subtitle (date/client, `text-off-black/40`)
  - Selected item: `bg-peach-100 border-l-4 border-primary`
  - Hover: `bg-peach-50`
  - Divider: `border-b border-border` between items
- FAB button: `+` circle button at top-right of header area, `bg-primary text-white rounded-full w-8 h-8`

### Detail Panel (Right)
- Flex-1, scrollable
- Top bar: `← Back` link (left), `Edit ↗` link (right)
- Entity name: `text-2xl font-bold`
- Status badge: icon + label in pill, `bg-peach-50 text-primary rounded-full px-3 py-1 text-sm`
- Action buttons: circular icon buttons top-right (phone, settings, email) — `border border-border rounded-full w-10 h-10`
- Tab strip: horizontal tabs below name — `border-b border-border`, active tab = `text-primary border-b-2 border-primary`
- Info banner: full-width, `bg-peach-50 border-l-4 border-primary px-4 py-3 text-sm`, with info icon

### Field Grid (Inside Detail Panel)
- Section title: `text-xl font-bold mt-8 mb-4`
- Fields arranged in 2-3 column grid: `grid grid-cols-3 gap-6`
- Each field: bold label (`text-sm font-bold text-off-black`) above value (`text-sm text-off-black mt-1`)
- Text areas: `bg-peach-50/50 rounded-lg p-4` for notes/descriptions

### Detail Panel Footer
- Sticky or absolute bottom
- Left: destructive action (`🗑 Delete` — outlined, burnt orange icon + text)
- Right: primary CTA button (`bg-primary text-white rounded-lg px-6 py-2`)

## 5. Page-Specific Mappings

### Dashboard → Overview with KPI cards + Activity Feed
- No master-detail. Full-width content area.
- KPI row: 3-4 metric cards in horizontal grid
  - Each card: white with `border border-border rounded-lg p-5`
  - Label (sm, medium weight) → Value (2xl, bold) → Subtitle (xs, muted)
- Activity feed: chronological timeline with colored dot indicators
  - Each entry: dot (colored circle, 8px) + timestamp + description
  - Dot colors: peach-100 for normal, primary for actions, peach-800 for alerts
- Recent projects: table or card list below

### Projects → Master-Detail
- List panel: project name + client name + date + budget amount
- Filter pills: All / Active / Completed / Pending
- Detail panel tabs: Overview / Budget / Transactions / Change Orders / Documents
- Overview tab: field grid with project info (client, timeline, total budget, funding method)
- Budget tab: category breakdown table with progress bars per category
- Transactions tab: table with columns — Date, Merchant, Category, Amount, Balance
- Change Orders tab: list of change orders with status badges

### Clients → Master-Detail
- List panel: client name + contact date + project count
- Filter pills: All / Homeowners / Property Managers / Business
- Detail panel tabs: Details / Projects / Activity
- Details tab: contact info field grid (name, email, phone, address)
- Projects tab: list of associated projects with status + budget
- Activity tab: timeline of interactions

### Transactions → Full-Width Table
- No master-detail. Full-width table view.
- Page title + count + filter pills (All / This Week / This Month / Flagged)
- Search bar right-aligned
- Table columns: Date, Project, Merchant, Category, Amount, Receipt, Status
- Alternating row backgrounds: white / `bg-peach-50/30`
- Sort indicators on column headers

### Virtual Cards → Card Display
- Card stack or grid showing issued virtual cards
- Each card: dark gradient background, masked card number, project name, spending limit
- Below each card: project name, budget utilization progress bar, status

## 6. Typography

### Font Stack
- **Headlines & Display:** Manrope (weights: 600, 700, 800)
- **Body & UI:** DM Sans (weights: 400, 500, 600, 700)

### Scale

| Element | Classes | Notes |
|---------|---------|-------|
| Page title (in list panel) | `text-2xl font-bold font-headline` | Large, bold |
| Entity name (detail panel) | `text-2xl font-bold font-headline` | Same scale as page title |
| Section title | `text-xl font-bold font-headline` | Within detail panel sections |
| Tab label | `text-sm font-medium` | Active: `text-primary`, inactive: `text-off-black/60` |
| Field label | `text-sm font-bold text-off-black` | Always bold, above value |
| Field value | `text-sm text-off-black` | Normal weight |
| List item name | `text-sm font-semibold` | In left list panel |
| List item subtitle | `text-xs text-off-black/40` | Date, client, etc. |
| KPI label | `text-sm font-medium text-off-black` | NOT uppercase |
| KPI value | `text-2xl font-bold tracking-tight font-headline` | Manrope for monetary amounts |
| Filter pill | `text-sm` | In pill buttons |
| Body text | `text-sm` | DM Sans |
| Minimum size | 10px | Never smaller |

### Typography Rules
- All monetary amounts use `font-headline` (Manrope) regardless of size.
- Field labels are always bold. This is a key Twilio CRM pattern — bold label, normal value.
- Section headers are plain bold text. NO icons next to section titles.
- Tab labels are medium weight, not bold. Active tab distinguished by color + bottom border.
- All text uses `text-off-black`. Sub-text/meta uses `text-off-black/40`.

## 10. Responsive Behavior

- **Mobile (< 768px):** Sidebar collapses to hidden (hamburger toggle). Master-detail becomes single-panel (list OR detail, not both). Full-width content.
- **Tablet (768px-1024px):** Sidebar collapsed (52px icons only). List panel narrows to 300px.
- **Desktop (> 1024px):** Full layout. Sidebar expandable. List panel 380px.
- Touch targets: minimum 44x44px on all interactive elements.

## 12. Information Density

This is a CRM for professionals managing real money. Screens should feel dense and useful, not spacious and decorative.

**Density rules:**
- List items: compact — `py-3` not `py-6`. Show 8-12 items in view without scrolling.
- Field grids: tight — `gap-y-4` not `gap-y-8`. Pack information, don't spread it out.
- Tables: compact rows — `py-2` to `py-3`. No oversized row heights.
- Sidebar: icons at 20px with 8px vertical gap. No oversized 48px icons with 24px gaps.
- Page title area: title + count on one line, filter pills immediately below. No hero sections, no giant top padding, no decorative subtitle text.
- Detail panel: sections start immediately after tabs. No spacer divs, no decorative headers.
- Modal: sized to content. `w-[420px]` max for confirmations. No full-screen overlays for simple actions.

**Test:** If you can see fewer than 5 data items on screen at once, the layout is too spacious. Tighten it.

## 13. Anti-Patterns (NEVER Do)

### Layout
- Grey canvas background — content area is white
- Shadow-elevation cards floating on grey — use bordered containers or flat sections
- Max-width constraints on content — content fills the available space
- Centered content layouts — left-aligned, full-width

### Color
- Blue accents anywhere — only peach/orange
- Green/red status indicators — use peach family
- Grey text for readable content — use `text-off-black`
- Multi-color card backgrounds — all white

### Components
- Icons next to section headers — plain bold text only
- Rounded-full buttons — use `rounded-lg`
- Shadow as primary card differentiator — use borders or flat sections within the white content area
- Uppercase labels — sentence case everywhere
- Decorative SVG blobs, gradient backgrounds, blur effects
- Generic dashboard card grids — use master-detail pattern

### Vibe-Coding / AI-Slop (absolute zero tolerance)
- Hero sections with taglines — this is a CRM, not a landing page
- Gradient blob backgrounds — no decorative backgrounds of any kind
- Glassmorphism / backdrop-blur / frosted glass effects
- Oversized padding (py-12, py-16, etc.) — keep density tight
- Centered single-column layouts — content is left-aligned and fills available space
- Generic illustrations or placeholder graphics
- "Modern SaaS" aesthetic with 80% whitespace and 20% content
- Decorative section dividers, wavy borders, or animated transitions
- Large rounded corners (rounded-2xl, rounded-3xl) — max is rounded-lg
- Emoji as UI elements
- Motivational copy ("Let's get started!", "Welcome back!") — show data, not greetings

### Patterns
- Single-panel list views without detail — always pair list with detail panel
- Tabs without bottom border indicator — active tab needs `border-b-2 border-primary`
- Filter buttons without pill shape — always `rounded-full` on filter pills
- Modals without overlay dimming — always `bg-black/50` overlay
- Forms without section grouping — group related fields under section titles

## 14. Coding Standards

### CSS Modules with @apply
All styling in CSS Module files (`.module.css`) using `@apply`. No inline Tailwind strings in JSX.

**Architecture:**
- `styles/shared.module.css` — shared patterns (list items, field grids, tabs, pills, modals)
- `app/<route>/<name>.module.css` — page-specific styles

**Import convention:**
```tsx
import s from "./projects.module.css";
import shared from "@/styles/shared.module.css";
import { cn } from "@/lib/utils";
```

### Server Components by Default
- Pages and layouts are Server Components
- Only leaf interactive components use `"use client"`
- Data fetching in Server Components, not `useEffect`

### Icon System
Use **Material Symbols Outlined** via `<Icon />` component exclusively.
- Default: FILL 0, wght 400, GRAD 0, opsz 24
- Active sidebar items: FILL 1
- Do NOT use lucide-react

### TypeScript
- `interface` for props, not `type`
- No `any` types
- Zod for runtime validation at API boundaries

### Semantic HTML
- `<main>`, `<nav>`, `<section>`, `<header>`, `<footer>` landmarks
- `<button>` for actions, `<Link>` for navigation
- 44x44px minimum touch targets
- 4.5:1 contrast ratio for text
